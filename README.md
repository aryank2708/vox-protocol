# Vox Protocol

Onchain identity, proof-of-call, and reputation/escrow rails for voice AI agents on **Monad testnet**.

> 350M underbanked people in India interact with financial voice agents — loan recovery, EMI reminders, credit onboarding. None of those calls are verifiable or portable. Disputes are unresolvable; payments need human reconciliation. Vox uses Monad as a trustless third party that fixes this.

## Monorepo layout

```
/contracts   Foundry — Solidity ^0.8.24
/backend     Express + ethers v6 — REST API to register agents, attest calls, release escrow
/frontend    Vite + React — dark-theme operator dashboard with live call simulation
/shared/abi  ABIs exported from forge build (consumed by backend + frontend)
```

## Smart contracts

- `AgentRegistry.sol` — register a voice agent: wallet, name, type (`COLLECTIONS | ONBOARDING | REMINDER | SUPPORT | OTHER`), authorized actions.
- `CallAttestation.sol` — log each call onchain: agent, transcript hash (keccak256), outcome (`PAID | PROMISED | NO_ANSWER | DNC`), duration, timestamp.
- `ReputationEscrow.sol` — merchant deposits MON; on each attested outcome, the contract auto-releases a configured payout to the agent operator and adjusts reputation (`PAID +10`, `PROMISED +4`, `NO_ANSWER -1`, `DNC -5`).

Default payouts (configurable by deployer):

| Outcome   | Payout (MON) |
|-----------|--------------|
| PAID      | 0.05         |
| PROMISED  | 0.02         |
| NO_ANSWER | 0.005        |
| DNC       | 0            |

## Quickstart

### 0. Install Foundry (once)

```bash
curl -L https://foundry.paradigm.xyz | bash && source ~/.zshenv && foundryup
```

### 1. Build & test contracts

```bash
cd contracts && forge build && forge test -vv
```

### 2. Deploy to Monad testnet

```bash
export PRIVATE_KEY=0x...                   # funded testnet key
cd contracts
forge script script/Deploy.s.sol \
  --rpc-url https://testnet-rpc.monad.xyz \
  --broadcast -vvv
```

Copy the printed `AgentRegistry`, `CallAttestation`, `ReputationEscrow` addresses.

### 3. Configure backend

```bash
cd ../backend
cp .env.example .env
# edit .env: paste PRIVATE_KEY + the three contract addresses
npm install
npm run extract-abi              # pulls ABIs from ../contracts/out into ./abi
npm run dev                      # http://localhost:8787
```

### 4. Start frontend

```bash
cd ../frontend
npm install
npm run dev                      # http://localhost:5173 (proxies /api -> :8787)
```

### 5. Demo flow

Open the dashboard. Click **Simulate Call**:

1. Backend registers a fresh demo agent on `AgentRegistry` (only first time).
2. Backend deposits 0.2 MON into `ReputationEscrow` as merchant (only if balance < 0.1).
3. Frontend plays a 30s hardcoded EMI-followup transcript live.
4. On call end, frontend POSTs `/attest-call` → backend writes `CallAttestation`.
5. Backend then POSTs `/release-call` → escrow auto-releases the `PROMISED` payout (0.02 MON) to the agent operator and bumps reputation by +4.
6. Dashboard auto-refreshes: identity card, call feed, escrow balance + release history. Each tx gets a clickable Monad explorer link.

## REST API

| Method | Path | Body / Query | Purpose |
|---|---|---|---|
| `GET`  | `/health` | — | provider check, signer, contract config |
| `GET`  | `/config` | — | chain + contract addresses for the frontend |
| `POST` | `/register-agent` | `{ wallet?, name, agentType, authorizedActions }` | register on `AgentRegistry` (auto-generates wallet if omitted) |
| `POST` | `/attest-call` | `{ agent, transcript?, transcriptHash?, outcome, duration }` | post `CallAttestation` |
| `POST` | `/release-call` | `{ callId, merchant }` | trigger escrow payout |
| `POST` | `/escrow/deposit` | `{ amount }` | merchant funds escrow (MON) |
| `GET`  | `/agent/:address` | — | profile + reputation + last 10 calls |
| `GET`  | `/calls/recent` | `?limit=10` | global call feed |
| `GET`  | `/escrow/state` | `?merchant=0x..` | balance + recent releases |
